# SB02 — EXTRAÇÃO DE FRAMEWORKS
> Fase 2 do Protocolo de Clonagem — Mapeamento dos modelos mentais e ferramentas de pensamento

---

## Prompt Principal

```
Você é um analista especializado em epistemologia aplicada — a ciência de como as pessoas pensam e estruturam seu conhecimento.

Com base na pesquisa realizada na Fase 1 sobre [NOME DA REFERÊNCIA], sua missão agora é extrair
os frameworks e modelos mentais que esta pessoa usa sistematicamente.

Um framework não é uma citação. É uma ferramenta de pensamento — um padrão repetível que a referência
aplica para analisar problemas, tomar decisões ou explicar fenômenos.

---

## BLOCO 1: FRAMEWORKS EXPLÍCITOS
(Ferramentas que a referência nomeia e ensina diretamente)

Para cada framework encontrado, preencha:

**Framework [X]: [NOME]**
- **Premissa central**: O que este framework assume como verdade?
- **Mecanismo**: Como funciona? Quais são os passos ou componentes?
- **Aplicação**: Em que tipo de situação esta ferramenta é usada?
- **Exemplo de uso**: Como a própria referência aplica este framework?
- **Limitação**: Onde este framework não funciona ou tem pontos cegos?
- **Origem**: De onde vem este framework? (Original, adaptado de outro pensador, etc.)

[Liste todos os frameworks explícitos que você identificou — idealmente 5-10]

---

## BLOCO 2: FRAMEWORKS IMPLÍCITOS
(Padrões de pensamento que a referência usa mas não nomeia como frameworks)

Aqui o trabalho é mais analítico. Identifique padrões repetíveis no raciocínio:

**Perguntas para identificar frameworks implícitos:**
- Que tipo de analogia esta referência usa sistematicamente?
- Que nível de análise ela prefere (individual/sistêmico/histórico)?
- Ela tende a raciocinar de exemplos para princípios ou de princípios para exemplos?
- Ela usa comparação histórica como método? Se sim, como?
- Ela tem preferência por análises de longo prazo vs. curto prazo?
- Ela privilegia dados quantitativos ou evidências qualitativas?
- Que campo de conhecimento externo ela importa mais frequentemente?

Para cada padrão identificado, dê um nome (pode ser seu) e explique o mecanismo.

---

## BLOCO 3: TAXONOMIAS E CATEGORIAS
(Os sistemas de classificação que a referência usa)

Muitos pensadores têm categorias próprias para organizar o mundo:
- "Tipo A" vs. "Tipo B" de empresa/pessoa/situação
- Fases ou estágios de um processo
- Matrizes 2x2 recorrentes
- Espectros ou escalas que ela usa para avaliar

Liste as taxonomias que esta referência usa. Para cada uma:
- O que está sendo classificado?
- Quais são as categorias?
- Qual é o critério de distinção entre elas?
- Por que esta classificação é útil no modelo dela?

---

## BLOCO 4: HEURÍSTICAS DE DECISÃO
(As regras de bolso que guiam como a referência decide)

Identifique as heurísticas — atalhos de raciocínio que emergem no trabalho dela:
- Princípios que ela repete em múltiplos contextos
- "Sempre que X, faça Y"
- Situações onde ela tem uma resposta padrão muito clara
- Coisas que ela considera invariavelmente erradas ou certas

---

## BLOCO 5: SÍNTESE — MAPA DE FRAMEWORKS
Organize tudo em uma tabela resumo:

| # | Nome do Framework | Tipo | Domínio de Aplicação | Frequência de Uso |
|---|------------------|------|---------------------|-------------------|
| 1 | [Nome] | Explícito/Implícito | [Estratégia/Criatividade/Liderança/etc] | Alta/Média/Baixa |
| 2 | [Nome] | ... | ... | ... |

---

## OUTPUT ESPERADO

1. **Lista de frameworks explícitos** com estrutura completa (Bloco 1)
2. **Frameworks implícitos identificados** com análise do padrão (Bloco 2)
3. **Taxonomias e categorias** usadas pela referência (Bloco 3)
4. **Heurísticas de decisão** recorrentes (Bloco 4)
5. **Mapa visual de frameworks** — tabela de síntese (Bloco 5)

Número mínimo de frameworks: 5
Número ideal: 8-12 (entre explícitos e implícitos)
```

---

## Prompt de Refinamento — Framework Específico

Use quando quiser aprofundar um framework específico já identificado:

```
Vamos aprofundar o [NOME DO FRAMEWORK] de [NOME DA REFERÊNCIA].

1. Qual é a origem deste framework? Ele surgiu de uma experiência específica?
2. Como a referência aplica este framework em diferentes contextos?
3. Há nuances ou variações do framework dependendo do contexto?
4. Que outras referências têm frameworks similares? O que os diferencia?
5. Como eu poderia aplicar este framework a [SITUAÇÃO ESPECÍFICA DO USUÁRIO]?
6. Quais são os sinais de que este framework está sendo mal aplicado?
7. O próprio framework evoluiu ao longo do tempo no trabalho desta referência?
```

---

## Prompt de Refinamento — Comparação Entre Frameworks

Use quando dois frameworks da mesma referência parecem se contradizer:

```
Na obra de [NOME DA REFERÊNCIA], parecem existir dois frameworks que tensionam entre si:
- [Framework A]: [descrição]
- [Framework B]: [descrição]

Ajude-me a entender:
1. Essa tensão é real ou aparente?
2. Se real, como a referência resolve isso? Ela tem uma meta-regra?
3. Se aparente, o que estou lendo errado?
4. Em que contextos cada framework se aplica?
5. Essa tensão é, ela mesma, reveladora de algo sobre o modelo mental desta referência?
```

---

## Critérios de Qualidade

- [ ] Pelo menos 5 frameworks identificados (explícitos + implícitos)
- [ ] Cada framework tem premissa, mecanismo e limitação mapeados
- [ ] Pelo menos 1 taxonomia ou sistema de categorias identificado
- [ ] Pelo menos 3 heurísticas de decisão listadas
- [ ] Tabela-síntese completa
